Bash Shell - Terminal - Command Line

In laymans terms, a shell means any program that is used to send commands to the operating system. A shell provides a means to interact with the kernel, and the 'lower levels' of the operating system.

If a software engineer wants to write a programme that requires the ability to interact with the underlying operating system (that would be all of them), for example to read/write from a file, or to play sound through the speakers, that is where a shell comes in handy. The author does not need to write the code from scratch, in fact in pretty much all OS's this is prohibitted, as these functions need to be standardised. The limited functions of the underlying operating system are all accessible through commands, which can be sent from a shell.

The Bash Shell is a Unix shell and command language written by Brian Fox. The terminal is the programme that provides the interface to the bash shell, allowing the user to interactively send commands directly to the OS. When we do this, we are interacting with the Command Line.

If you want to understand how a general computer application functions, you are usually going to need to understand how it is interacting with the OS, that is the fundamental reason as to why it is important to have the necessary knowledge of UNIX commands, to know what you can ask from your OS and how you can ask it.

If you are here because you want to be a penetration tester - then it is absolutely 100% necessary to be comfortable with the command line of your chosen OS.

Don't worry, the whole point of the bandit game is to get you comfortable using the command line. Right now generally you can think of the terminal programme as an extremely 'raw' and stripped back interface into the heart of your computer.

SSH - Secure Shell.

Since we will be playing the wargames remotely - much like in real life where the target computer is likely to be a remote system - we are going to need to have the ability to run a bash shell on the target computer (in this case the over the wire servers).

To do this, we will be utilising the power of SSH.

definition.... every command we type into our terminal will be sent to the connected server. as if we were physically sitting at the machine and running a shell (a command line/terminal session)

You can learn about SSH in more depth here.
